# Security Policy

## Supported Versions

PocketFrame is currently in alpha. Security fixes are only guaranteed for the
latest release on `main`.

## Reporting a Vulnerability

Please do not report security vulnerabilities in public issues.

Use GitHub private vulnerability reporting:

https://github.com/JeffDraws/PocketFrame/security/advisories/new

If private reporting is unavailable, open a normal issue with minimal detail
and request a private contact path from maintainers.

## What to include

- A short description of the issue and affected component
- Reproduction steps or proof-of-concept
- Potential impact
- Suggested mitigation (if known)

## Response target

Maintainers will aim to acknowledge reports within 5 business days and provide
status updates as triage progresses.
